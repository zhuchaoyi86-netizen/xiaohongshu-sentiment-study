## 当前目录结构

- `最终成果汇总/`：最终交付成果与核心数据结果，适合直接查看、提交和展示。
- `output/`：脚本运行产生的中间输出结果，主要用于数据处理和结果复现。
- `scripts/`：数据清洗、变量构建、回归分析和文档生成脚本。
- `过程归档/`：研究过程中的笔记、Notebook 和中间文档，默认不作为最终交付内容。
- `data/`：原始数据或后续补充数据预留目录。
- `python/`：课程原始学习资料，已与本项目主线区分。

## 仓库整理原则

- 以“最终成果”和“核心数据”作为主线内容。
- 过程性材料统一收入 `过程归档/`，避免与交付成果混在一起。
- 最终成果目录仅保留真正需要展示、提交和引用的文件。

## 后续使用建议

- 如果要继续修改研究内容，优先在 `scripts/` 和 `output/` 中操作。
- 如果要交作业、发老师或展示给同学，优先使用 `最终成果汇总/` 中的文件。
