# 情感变量划分方案

## 3.2 情感量化处理

本文重点关注小红书食品种草笔记中的正向与负向情绪表达，并分析其对点赞热度的影响。考虑到课程设计的可操作性与解释性，本文采用基于情感词识别的文本量化方法，对笔记正文进行情感划分。

在前期文本清洗基础上，本文使用清洗后的情感分析文本字段 `desc_sentiment` 作为情感识别对象。相较于原始正文，该字段已尽量剔除话题标签、用户提及、平台表情标记及无关符号，更能反映笔记作者真实的情绪表达。

具体而言，本文预先设定正向情绪词和负向情绪词，并统计每条文本中对应词语的出现情况。在此基础上，构造情感得分变量：

`sentiment_score = 正向词频 - 负向词频`

若情感得分大于 0，则判定该文本以正向情绪为主；若情感得分小于 0，则判定该文本以负向情绪为主；若情感得分等于 0，则视为中性或情绪不明显文本。

## 情感变量定义

建议至少构造下面 4 个变量：

| 变量名 | 含义 | 构造方式 |
|---|---|---|
| `sentiment_score` | 情感得分 | 正向词数减去负向词数 |
| `positive_flag` | 是否为正向情绪 | `sentiment_score > 0` 记为 1，否则为 0 |
| `negative_flag` | 是否为负向情绪 | `sentiment_score < 0` 记为 1，否则为 0 |
| `neutral_flag` | 是否为中性文本 | `sentiment_score = 0` 记为 1，否则为 0 |

如果你们老师更喜欢简单一点，也可以只保留两个核心变量：

- `positive_flag`
- `negative_flag`

这样更贴合“正负情绪对点赞热度影响”的题目表达。

## 变量解释方式

你在论文里可以这样解释：

“情感得分越高，说明文本整体情绪越积极；情感得分越低，说明文本整体情绪越消极。正向情绪变量用于识别推荐、喜爱、满意等积极表达，负向情绪变量用于识别失望、踩雷、不推荐等消极表达。”

## 为什么这样做

- 方法简单，适合课程设计
- 结果容易解释，老师一看就明白
- 能直接进入后续回归模型
- 和你的题目“情感对点赞热度的影响”匹配度高

## 后续模型里的放法

你后面第四部分可以这样设：

- 因变量：`log_likes`
- 核心自变量：`sentiment_score` 或 `positive_flag`、`negative_flag`
- 辅助控制变量：`text_length`、`has_image`、`imagenumber`、`time_period`

## 可直接放进论文的简短表述

本文基于清洗后的食品种草笔记文本，采用情感词识别法对正文进行情绪量化处理。具体做法是预先设定正向和负向情感词典，并统计每条文本中的相关词频，以“正向词频减去负向词频”的方式构造情感得分变量。进一步地，本文将情感得分大于 0 的文本定义为正向情绪文本，将情感得分小于 0 的文本定义为负向情绪文本，将情感得分等于 0 的文本定义为中性文本。该处理方式能够将原始文本信息转化为可进入计量模型的结构化变量，为后续检验情感表达对点赞热度的影响提供基础。
