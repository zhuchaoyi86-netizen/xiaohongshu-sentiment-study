# 第三部分 数据预处理与变量构建

## 3.1 原始数据清洗流程

在原始小红书评论数据中，可能存在空值、重复值以及与研究目标无关的低质量文本。如果不先进行清洗，后续情感分析和实证建模的结果会受到明显干扰。因此，本研究首先对原始数据进行了系统的数据预处理。

首先，对评论文本为空、仅包含空格或无实际语义内容的记录进行了删除，以保证后续文本分析对象具有基本信息价值。其次，对完全重复的评论记录进行了去重处理，避免同一条评论被重复计入样本。再次，针对明显灌水、机械重复、字符过少或仅由符号组成的评论，结合文本长度和内容特征进行了识别与剔除。经过上述处理后，数据质量得到明显提升，为后续变量构建和模型分析奠定了基础。

## 3.2 文本情感量化处理

在文本分析部分，本研究对评论内容进行了情感量化处理，目的是将原本难以直接进入计量模型的文本信息转化为可计算的数值变量。具体而言，可以采用情感词典法或现成中文情感分析工具，对每条评论生成情感倾向得分。情感得分越高，表示评论整体情绪越积极；得分越低，则表示评论情绪越消极。

采用情感量化处理有两方面意义：一是能够更直观地描述评论区用户态度分布特征；二是能够将情感变量纳入后续模型，用于检验情绪表达与点赞量之间是否存在显著关系。

如果老师没有强制要求复杂方法，你就写：

“本文采用基于中文情感分析工具的情感打分方法，对评论文本进行逐条测算，并将输出结果标准化为可比较的情感得分变量。”

## 3.3 衍生变量构建

为了更全面地刻画小红书评论的内容特征与互动特征，本文在原始字段基础上进一步构建了若干衍生变量。

第一，构建文本长度变量，以评论文本的字符数衡量评论内容丰富程度。一般而言，文本越长，所传递的信息越多，可能更容易引发其他用户互动。

第二，构建表情变量。若评论文本中包含表情符号，则赋值为 1，否则赋值为 0。该变量用于考察情绪表达方式对互动表现的影响。

第三，构建配图相关变量。若原始数据中包含图片数量或是否附图的信息，可将其转化为是否配图的虚拟变量，用于衡量多模态表达对用户反馈的作用。

第四，构建时间时段变量。根据评论发布时间，将其划分为凌晨、上午、下午、晚上等不同时间区间，用于分析不同发布时间段下用户互动差异。

| 变量名称 | 变量含义 | 构建方式 |
|---|---|---|

| time_period | 发布时间段 | 按小时划分为不同区间 |
| sentiment_score | 情感得分 | 情感分析工具输出 |

## 3.4 点赞量对数变换与数据优化

在评论互动指标中，点赞量通常呈现明显的右偏分布，即少数评论点赞量很高，而大多数评论点赞量较低。这种分布特征容易导致模型估计受极端值影响。因此，本文对点赞量变量进行对数变换处理，即使用 ln(likes + 1) 作为新的因变量，以减弱异常值冲击并提高数据分布的平稳性。

此外，为提升后续模型估计的稳健性，本文还对部分极端异常值进行了识别，并结合样本分布特征进行合理处理。经过对数变换和优化后，数据更适合进入后续的回归分析与可视化展示。

如果老师问为什么加 1，你就答：

“因为原始点赞量中可能存在 0 值，直接取对数会无定义，因此通常使用 ln(x+1) 处理。”
